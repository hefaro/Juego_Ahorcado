Para que el desarrollo sea exitoso y no tengamos errores de integración, he diseñado este Protocolo de Estructura y Metodología. Este documento servirá como el "plano" para construir el sistema completo de forma profesional y escalable.📐 Estructura del Sistema: "Ahorcado Matemático Sincronizado"1. Arquitectura de Datos (Firebase)Para permitir que cientos de estudiantes jueguen desde sus casas o en el salón simultáneamente, los datos se organizarán por Salas Dinámicas.Ruta de Juego: sesiones/{nombre_de_la_sala}/palabra_actual: La ecuación o término oculto.letras_adivinadas: Array con los intentos correctos.errores: Contador (0 a 6).turno_de: Identificador del equipo que debe responder.estado: "jugando", "victoria" o "derrota".2. Metodología de Interacción (Dos Interfaces)El sistema se dividirá en dos archivos HTML independientes que se comunican a través de tu Firebase:Interfaz A (El Tablero): Pantalla visualmente atractiva (para proyectar o ver en pantalla grande) que muestra el dibujo del ahorcado y la palabra.Interfaz B (El Control): Pantalla optimizada para móviles (Mobile-first) donde el equipo ingresa las letras. Incluye un "Muro de Validación": antes de enviar una letra, el grupo debe resolver un reto matemático aleatorio.📜 Reglas de Juego (El Algoritmo de Competencia)Ingreso por Sala: Al abrir el juego, los jugadores deben escribir un "Nombre de Sala" (ej. 8A-Equipos). Todos los que escriban el mismo nombre verán el mismo juego.Validación Cognitiva: Para activar el teclado de letras, el sistema mostrará un ejercicio (ej: "Resuelve: $2x + 5 = 11$"). Solo si la respuesta es correcta ($x=3$), el equipo puede proponer una letra.Persistencia del Turno:Si la letra está en la palabra: El equipo mantiene el turno y suma puntos parciales.Si la letra no está: Se dibuja una parte del ahorcado y el turno pasa automáticamente al oponente.Finalización y Registro: Al ganar o perder, el sistema invoca automáticamente tu función registrarResultado para guardar el desempeño en el historial general de la institución.🛠️ Requerimientos Técnicos del Prompt de DesarrolloLenguaje: HTML5, CSS3 (Flexbox para diseño móvil) y JavaScript Moderno (ES6+).Conexión: Importación de Firebase mediante CDN (versión 10.10.0) para compatibilidad con GitHub Pages.Entrega: Se deben entregar archivos completos (no fragmentos) para evitar errores de implementación por parte del usuario.Contenido Pedagógico: El banco de palabras y retos debe centrarse en los currículos de 8° y 9° grado (Álgebra, Geometría y Estadística).
